📱 스마트폰(Termux) 개인 클라우드 완벽 구축 가이드

💡 프로젝트 개요

이 가이드는 사용하지 않는 안드로이드 기기(예: 삼성 갤럭시 노트 10+, Android 버전 12)를 활용하여 Nginx + PHP + MariaDB 기반의 강력한 Nextcloud 서버를 구축하는 상세 문서입니다. 모든 운영체제 시스템은 스마트폰의 고속 내부 메모리에 구동되며, 대용량 사용자 데이터(사진, 파일)는 마이크로 SD 카드에 분리 저장하는 '이중 스토리지 아키텍처'를 채택했습니다.

[1단계] 스마트폰 완전 초기화 및 기초 공사

💡 이 단계의 목표

기기를 공장초기화하여 깨끗한 상태를 만들고, 24시간 서버가 중단되지 않도록 전원 설정을 변경합니다. 가장 중요한 핵심은 노트북에서 스마트폰으로 원격 접속(SSH)하여 편하게 명령어를 복사/붙여넣기 할 수 있는 환경을 가장 먼저 구축하는 것입니다.

1. 기기 공장초기화 및 기본 설정

이전의 잘못된 설정이나 캐시 파일이 서버 구동을 방해하지 않도록 스마트폰을 완전히 초기화합니다.

  1. 스마트폰 [설정] > [일반] > [초기화] > [기기 전체 초기화]를 진행합니다.
  2. 초기화 완료 후 와이파이를 연결하고, 불필요한 기본 앱들은 모두 사용 중지 또는 삭제합니다.

2. 서버 생존을 위한 '배터리 최적화' 해제 (★매우 중요)

안드로이드 운영체제는 화면이 꺼지면 배터리를 아끼기 위해 앱을 강제로 종료합니다. 서버가 죽는 것을 막으려면 이 기능을 반드시 해제해야 합니다.

⚙️ 설정 방법
  • [설정] > [애플리케이션] > [Termux] > [배터리]로 이동하여 제한 없음으로 설정합니다.
  • 디스플레이 설정에서 '화면 자동 꺼짐 시간'을 최대한 길게 설정해 둡니다.

3. F-Droid 및 Termux 설치

Google Play 스토어 버전은 오류를 유발하므로 반드시 오픈소스 마켓인 F-Droid를 사용합니다.

  1. 스마트폰 브라우저에서 F-Droid 홈페이지에 접속하여 F-Droid.apk를 다운로드하고 설치합니다.
  2. F-Droid 앱을 실행하고 Termux를 검색하여 설치합니다.

4. 저장소 권한 부여 및 시스템 최신화

이제 스마트폰에서 Termux 앱을 열고 아래 명령어들을 직접 타건합니다.

# 안드로이드 저장소 접근 권한 요청 (팝업 창에서 '허용' 클릭)
termux-setup-storage

# 시스템 전체 의존성 갱신 (진행 중 Y/N을 물으면 Y 입력 후 엔터)
pkg update && pkg full-upgrade -y

5. PC 원격 접속(SSH) 환경 구축 (게임 체인저!)

긴 명령어를 스마트폰에서 치는 것은 오타의 원인이 됩니다. 동일한 와이파이에 연결된 노트북(PC)에서 스마트폰으로 접속하여 쾌적하게 작업할 수 있도록 SSH 서버를 엽니다.

📱 스마트폰(Termux)에서 할 일

1) SSH 패키지를 설치합니다.

pkg install openssh -y

2) 접속에 사용할 비밀번호를 설정합니다.

passwd

3) 내 스마트폰의 IP 주소를 확인합니다.

ifconfig

4) SSH 서버 엔진을 켭니다.

sshd
💻 노트북(PC)에서 접속하기

이제 스마트폰은 충전기에 꽂아 옆에 두시고, 편안하게 노트북 키보드 앞에 앉으세요!

  1. 노트북에서 명령 프롬프트(CMD) 또는 Windows PowerShell을 실행합니다.
  2. 아래 형식에 맞춰 접속 명령어를 입력합니다 (Termux의 SSH 기본 포트는 22가 아닌 8022입니다).
# 명령어 형식: ssh -p 8022 [스마트폰의 IP 주소]
# 예시:
ssh -p 8022 192.168.0.15

Are you sure you want to continue connecting? 메시지가 나오면 yes를 치고, 아까 스마트폰에서 설정한 비밀번호를 입력하면 접속 성공입니다!

6. 기초 필수 유틸리티 개별 설치 (PC에서 진행)

노트북과 연결된 터미널 창에서 아래 명령어들을 마우스로 편하게 복사 ➡️ 붙여넣기 하며 하나씩 설치합니다.

# 웹에서 파일을 다운로드하는 도구
pkg install wget -y

# 네트워크 통신 테스트 도구
pkg install curl -y

# 터미널용 텍스트 편집기
pkg install nano -y

# 압축 해제 도구
pkg install tar -y

# 보안 인증서 관련 도구
pkg install openssl -y
✅ 1단계 완료!

축하합니다! 오타 걱정 없이 마우스 클릭만으로 서버를 지을 수 있는 완벽한 원격 제어 베이스캠프가 완성되었습니다. 다음 단계에서는 이 쾌적한 노트북 환경에서 웹 서버(Nginx)와 데이터베이스(MariaDB)를 설치해 보겠습니다.

[2단계] 핵심 엔진 설치 및 데이터베이스 무결점 초기화

💻 PC 원격 접속(SSH) 상태에서 진행하세요!

1단계에서 구축한 SSH 환경을 통해 노트북의 터미널 창에서 작업합니다. 스마트폰의 작은 화면 대신, 넓은 모니터에서 아래 명령어들을 마우스로 복사 ➡️ 붙여넣기 하여 쾌적하게 서버를 구축해 보세요.

1. Nextcloud 구동용 핵심 패키지 개별 설치

웹 서버 역할을 하는 Nginx, 데이터를 처리하는 PHP, 데이터를 저장하는 MariaDB를 설치합니다. 의존성 꼬임으로 인한 설치 실패를 막기 위해 반드시 한 줄씩 따로따로 복사하여 설치합니다.

# 웹 서버
pkg install nginx -y

# 데이터베이스
pkg install mariadb -y

# PHP 엔진
pkg install php-fpm -y

2. PHP 엔진 최적화 (타임아웃 에러 원천 차단)

스마트폰 AP 특성상 연산 시간이 PC보다 길어질 수 있습니다. 기본 설정(30초)을 그대로 두면 Nextcloud 설치 시 504 Gateway Time-out 에러가 발생하므로, 엔진의 '대기 시간'을 1시간(3600초)으로 대폭 늘려주고 필수 모듈을 활성화합니다.

🔧 PHP 설정 파일 원격 수정

아래 명령어 네 줄을 차례대로 복사해서 실행하면, 설정 파일(php.ini)을 직접 열지 않고도 필요한 부분만 완벽하게 덮어쓸 수 있습니다.

# 최대 실행 시간 및 입력 대기 시간 3600초로 연장
sed -i 's/max_execution_time = .*/max_execution_time = 3600/' $PREFIX/lib/php.ini
echo "max_input_time = 3600" >> $PREFIX/lib/php.ini

# MariaDB와의 원활한 통신을 위한 필수 확장 모듈 활성화
echo "extension=pdo_mysql" >> $PREFIX/lib/php.ini
echo "extension=mysqli" >> $PREFIX/lib/php.ini

3. 데이터베이스 엔진 가동 및 초기화

MariaDB를 처음으로 켜고, Nextcloud가 사용할 수 있는 '방'을 만들어 주는 단계입니다.

# 백그라운드에서 MariaDB 가동 (명령어 끝의 & 기호 필수)
mysqld_safe &

# 엔진이 완전히 켜질 때까지 5초 대기
sleep 5

4. 계정 생성 및 권한 부여 (무결점 버전)

Nextcloud 전용 데이터베이스를 생성하고, 해당 DB를 관리할 사용자(db_nc_user)를 만들어 권한을 부여합니다. 꼬여있는 기존 데이터를 완전히 지우고 깨끗하게 시작하는 무결점 명령어입니다.

# 1. (선택 사항) 만약 꼬인 데이터가 있다면 방 비우기
mariadb -u root -e "DROP DATABASE IF EXISTS nextcloud;"

# 2. 깨끗한 Nextcloud 전용 데이터베이스 생성
mariadb -u root -e "CREATE DATABASE nextcloud;"

# 3. 데이터베이스 관리자 계정(db_nc_user) 생성 및 비밀번호(db_password1234) 지정
mariadb -u root -e "GRANT ALL PRIVILEGES ON nextcloud.* TO 'db_nc_user'@'localhost' IDENTIFIED BY 'db_password1234';"

# 4. 권한 즉시 적용
mariadb -u root -e "FLUSH PRIVILEGES;"
✅ 2단계 완료!

웹 서버와 PHP 엔진이 준비되었고, 데이터를 담을 그릇(데이터베이스)까지 완벽하고 깨끗하게 생성되었습니다. 다음 단계에서는 본격적으로 DuckDNS를 통한 외부 도메인 연동 및 완벽한 Nginx 설정을 진행해 보겠습니다.

[3단계] Nextcloud 다운로드 및 SD카드 전용 폴더 구축

💡 기업형 '이중 스토리지(투 트랙)' 아키텍처 구현

스마트폰의 내부 메모리는 시스템을 돌리는 데 특화되어 속도가 매우 빠르지만 용량이 적습니다. 반대로 SD 카드는 속도는 평범하지만 저렴하게 대용량을 구축할 수 있습니다. 이 단계에서는 Nextcloud의 엔진은 스마트폰 내장 메모리(Home)에 두고, 용량을 많이 차지하는 사진과 영상은 SD 카드로 직행하도록 분리 설계를 진행합니다.

1. Nextcloud 서버 본체 다운로드 및 압축 해제

먼저 공식 서버에서 설치 파일을 스마트폰 내부 메모리로 다운로드합니다. PC의 터미널(SSH) 창에 아래 명령어를 입력합니다.

# 사용자의 홈 디렉토리로 이동
cd ~

# 공식 서버에서 최신 버전 다운로드
wget https://download.nextcloud.com/server/releases/latest.tar.bz2

# 압축 해제 (글씨가 빠르게 올라가며 다소 시간이 소요됨)
tar -xjf latest.tar.bz2

# 설치가 끝난 원본 파일 삭제 (용량 확보)
rm latest.tar.bz2

2. 마이크로 SD 카드에 데이터 전용 폴더 생성

안드로이드 보안 정책상, Termux가 SD 카드에 데이터를 저장하려면 반드시 정해진 전용 경로(Android/data/com.termux/files)를 사용해야 합니다.

⚠️ 매우 중요: 내 SD 카드 고유 번호 찾기

아래 명령어에 쓰인 2263-EEDB는 단순한 예시입니다. SD 카드의 8자리 고유 번호는 스마트폰이나 카드마다 무조건 다릅니다. 반드시 본인의 번호로 수정해서 명령어를 입력해야 합니다.

[권한 오류 없이 내 번호 확인하는 법]
  • 안드로이드 최신 버전에서는 ls /storage 명령어가 'Permission denied(권한 없음)'으로 막히는 경우가 많습니다.
  • 대신 터미널에 df -h 명령어를 입력해 보세요. 출력되는 목록 중에서 맨 오른쪽에 /storage/XXXX-XXXX 형태(예: 1A2B-3C4D)로 마운트된 경로가 바로 본인의 SD 카드입니다.
# 본인의 SD카드 번호로 수정하여 입력!
# SD 카드 내부에 Nextcloud 데이터 전용 폴더(nc_data) 생성
mkdir -p /storage/2263-EEDB/Android/data/com.termux/files/nc_data

3. 저장소 인식용 필수 파일 생성 및 보안 설정

Nextcloud는 지정된 폴더 안에 .ocdata라는 빈 파일이 있어야만 정상적인 저장소로 인정합니다. 또한 외부 앱이 접근하지 못하도록 권한을 잠가야 합니다.

# 저장소 인증을 위한 .ocdata 숨김 파일 생성
touch /storage/2263-EEDB/Android/data/com.termux/files/nc_data/.ocdata

# 외부 접근을 막기 위해 폴더 권한을 770으로 제한
chmod -R 770 /storage/2263-EEDB/Android/data/com.termux/files/nc_data
✅ 3단계 완료!

축하합니다! 서버의 엔진과 대용량 데이터 창고(SD카드)가 모두 완벽하게 분리되어 세팅되었습니다. 다음 단계에서는 이 둘을 이어주고 외부에서 안전하게 접속할 수 있도록 DuckDNS 외부 도메인 연동 및 완벽한 Nginx 보안(SSL) 설정을 진행해 보겠습니다.

[4단계] DuckDNS 연동 및 Nginx 무결점 보안(SSL) 설정

💡 포트포워딩을 위한 필수 보안 과정

집 밖에서도 내 클라우드에 접속하려면 주소가 필요합니다. 가정용 인터넷은 IP가 수시로 바뀌므로 무료 도메인 서비스인 DuckDNS를 이용해 고정된 주소를 만들고, Let's Encrypt를 통해 군사급 암호화 통신(HTTPS)이 가능하도록 자물쇠를 채우는 과정입니다.

1. acme.sh 설치 및 인증 환경 구성

인증서를 무료로 발급받고 자동으로 갱신해 주는 똑똑한 스크립트인 acme.sh를 설치합니다. PC 원격(SSH) 터미널에 아래 명령어를 입력하세요.

# acme.sh 다운로드 및 설치
curl https://get.acme.sh | sh

# 설치된 스크립트를 현재 터미널에 바로 적용
source ~/.bashrc

2. DuckDNS 토큰 입력 및 SSL 인증서 발급

사전에 DuckDNS 홈페이지(duckdns.org)에서 본인만의 도메인(예: 본인도메인.duckdns.org)을 만들고, 화면에 보이는 Token(토큰) 값을 복사해 두어야 합니다.

⚠️ 도메인 이름과 토큰 수정 필수

아래 명령어에서 발급받은_토큰_값_입력 부분과 본인도메인.duckdns.org 부분을 반드시 본인의 정보로 변경하여 실행하세요.

# DuckDNS 토큰 등록 (서버가 내가 도메인 주인임을 증명하는 과정)
export DuckDNS_Token="발급받은_토큰_값_입력"

# ECC 방식의 빠르고 가벼운 SSL 인증서 발급 요청
acme.sh --issue --dns dns_duckdns -d 본인도메인.duckdns.org

3. Nginx 설정 파일 교체 (무결점 마스터 버전)

이 단계는 전체 서버 구축 과정 중 가장 오류가 잦은 마의 구간입니다. 일반적인 가이드대로 설정하면 접속 시 화면이 하얗게 변하며 500 Internal Server Error가 뜨거나, Nginx가 rewrite or internal redirection cycle(무한 루프)에 빠져버립니다.

🔧 트러블슈팅 완벽 반영

아래 제공되는 설정 코드는 무한 루프 에러를 해결하고, 구버전의 listen 58080 ssl http2; 문법 경고를 없애기 위해 http2 on;을 분리했으며, PHP 대기 시간을 3600초로 늘려 타임아웃을 방지한 '스마트폰 완벽 호환 버전'입니다.

# 기존에 꼬여있거나 기본으로 제공되는 Nginx 설정 파일 삭제
rm $PREFIX/etc/nginx/conf.d/nextcloud.conf

🔧 트러블슈팅: OPcache 경고가 계속 뜰 때

# 환경 변수(getenv) 읽기 권한 허용
sed -i 's/;clear_env = no/clear_env = no/' $PREFIX/etc/php-fpm.d/www.conf

opcache.enable=1
opcache.enable_cli=1
opcache.memory_consumption=128
opcache.interned_strings_buffer=16
opcache.max_accelerated_files=10000
opcache.revalidate_freq=1
opcache.save_comments=1
EOF

🔧 트러블슈팅: .mjs 파일 인식 오류 완벽 해결

MIME 타입에 mjs를 추가하더라도, Nginx 라우팅 규칙(정규식)에 mjs가 빠져있으면 에러가 사라지지 않습니다. 아래 명령어 2개로 양쪽 모두를 완벽하게 뚫어줍니다.

# 1. MIME 타입 목록에 mjs 확장자 인식 추가
sed -i 's/application\/javascript.*js;/application\/javascript js mjs;/' $PREFIX/etc/nginx/mime.types

# 2. Nginx 설정 파일(nextcloud.conf)의 정적 파일 처리 규칙(정규식)에 mjs 강제 추가
sed -i 's/css|js|svg/css|js|mjs|svg/' $PREFIX/etc/nginx/conf.d/nextcloud.conf

이어서 Nginx 설정 파일(nextcloud.conf)을 nano $PREFIX/etc/nginx/conf.d/nextcloud.conf 명령어로 열고, server_name 아래에 다음 3줄을 수동으로 추가하여 HSTS 보안과 캘린더 동기화 주소 문제를 해결합니다.

add_header Strict-Transport-Security "max-age=15552000; includeSubDomains" always;
location = /.well-known/carddav { return 301 $scheme://$host:$server_port/remote.php/dav; }
location = /.well-known/caldav  { return 301 $scheme://$host:$server_port/remote.php/dav; }

💡 포트포워딩이란?

외부망(인터넷)에서 우리 집 공유기로 들어오는 신호를 집 안에 있는 특정 기기(스마트폰 서버)로 전달해주는 이정표 역할을 합니다. 이 설정을 하지 않으면 외부에서 내 클라우드 주소를 쳐도 공유기 문턱에서 막히게 됩니다.

1. 공유기 관리 페이지 접속

  • ipTIME: 192.168.0.1
  • TP-Link: 192.168.0.1 또는 192.168.1.1
  • ASUS: 192.168.1.1 또는 router.asus.com
  • KT/SKT/LG 통신사 공유기: 공유기 뒷면 스티커에 적힌 주소 확인

2. 포트포워딩 규칙 추가

관리 페이지 내 [고급 설정] > [네트워크 관리] > [포트포워드 설정] 메뉴로 이동하여 새로운 규칙을 추가합니다.

[9단계] Windows 노트북 동기화 클라이언트 설치 및 파일 연동

💡 나만의 드라이브(내 PC) 만들기

스마트폰 서버에 매번 웹 브라우저로 접속해서 파일을 올릴 필요가 없습니다. 삼성 노트북(NT550P5C-S55S)에 Nextcloud 공식 프로그램을 설치하면, 윈도우 탐색기 안에 전용 폴더가 생기며 이 폴더에 파일을 넣기만 하면 자동으로 스마트폰(SD카드) 서버로 전송되고 동기화됩니다.

1. PC 클라이언트 다운로드 및 설치

먼저 노트북에서 Nextcloud 공식 홈페이지에 접속하여 윈도우용 설치 파일을 다운로드합니다.

https://nextcloud.com/install/#install-clients
  1. 위 주소로 접속하여 Download for Desktop 버튼을 누르고 Windows 버전을 클릭해 다운로드합니다.
  2. 다운로드된 설치 파일(Nextcloud-x.x.x-x64.msi)을 실행하여 기본 설정 그대로 [Next]를 눌러 설치를 완료합니다.
  3. 설치가 끝나고 프로그램이 실행되면 컴퓨터를 한 번 재부팅 할지 묻는 창이 나올 수 있습니다. 안정적인 연동을 위해 가급적 재부팅을 권장합니다.

2. 서버 주소 입력 및 로그인 (접근 권한 허용)

프로그램이 실행되면 방금까지 우리가 구축한 노트10+ 스마트폰 서버와 연결하는 과정을 거칩니다.

  1. 첫 화면에서 [Log in (로그인)] 버튼을 클릭합니다.
  2. 서버 주소(Server Address) 입력창이 나오면 아래와 같이 포트 번호(58080)까지 포함된 완벽한 외부 접속 주소를 입력하고 [Next]를 누릅니다.
⚠️ 서버 주소 입력 주의사항

반드시 https:// 로 시작해야 하며, 끝에 :58080 포트를 빼먹으면 연결되지 않습니다.

https://본인도메인.duckdns.org:58080
  1. 노트북의 기본 웹 브라우저가 열리며 Nextcloud 로그인 화면이 나타납니다. 이전에 만들었던 본인의 관리자 아이디와 비밀번호를 입력합니다.
  2. [접근 허용 (Grant access)] 버튼을 클릭하여 PC 프로그램이 서버와 통신할 수 있도록 권한을 부여합니다.

3. 동기화 방식 및 로컬 폴더 설정

로그인이 완료되면 PC의 어느 폴더를 서버와 똑같이 맞출 것인지(동기화) 설정하는 창이 나타납니다. 여기서 아주 중요한 선택을 해야 합니다.

💡 가상 파일(Virtual Files) vs 전체 동기화
  • 가상 파일 사용 (권장): 서버에 있는 파일 목록(껍데기)만 PC 탐색기에 보여주고, 실제로 파일을 더블클릭해서 열 때만 다운로드합니다. 삼성 노트북의 하드디스크 용량을 완벽하게 아낄 수 있는 최고의 기능입니다.
  • 모든 파일 동기화 (기존 방식): 서버에 100GB의 파일이 있다면 노트북 하드디스크에도 똑같이 100GB 공간을 차지하며 즉시 다운로드해 둡니다. 인터넷이 안 되는 오프라인 환경에서 작업할 일이 많을 때 유리합니다.
  1. 원하는 동기화 방식(가상 파일 권장)을 선택합니다.
  2. 로컬 폴더(Local Folder): 윈도우 탐색기에서 Nextcloud 전용 폴더가 생성될 위치를 지정합니다. 기본값(C:\Users\사용자이름\Nextcloud)을 그대로 사용해도 좋습니다.
  3. [연결 (Connect)] 버튼을 클릭하면 세팅이 완료됩니다!

4. 파일 전송 테스트 (탐색기 연동 확인)

모든 연결이 끝났습니다. 이제 실제로 파일이 잘 오가는지 확인해 볼 차례입니다.

  1. 윈도우 바탕화면 하단 작업 표시줄이나 윈도우 탐색기(내 PC)의 왼쪽 즐겨찾기 메뉴를 보면 파란색 동그라미의 Nextcloud 아이콘이 새로 생긴 것을 볼 수 있습니다.
  2. 해당 폴더를 열고, 테스트용 사진이나 텍스트 파일을 하나 복사해서 폴더 안에 붙여넣어 봅니다.
  3. 파일 아이콘에 파란색 뱅글뱅글 도는 화살표(동기화 중)가 나타났다가 초록색 체크 마크(동기화 완료)로 변하면 노트10+ 서버(SD카드)로 완벽하게 전송된 것입니다!
  4. 이제 웹 브라우저로 서버에 접속해 보면, 방금 노트북에서 넣은 파일이 웹에도 똑같이 들어와 있는 것을 눈으로 확인할 수 있습니다.

🚀 진정한 개인 업무 클라우드 환경 완성!

수고하셨습니다! 이제 USB 메모리나 거추장스러운 케이블 연결 없이도, 노트북 탐색기에 파일을 쓱 밀어 넣으면 스마트폰 서버에 영구 저장되고 전 세계 어디서든 꺼내볼 수 있습니다. 추가로 스마트폰(본 기기) 플레이스토어에서 Nextcloud 앱을 설치하시면 스마트폰으로 찍은 사진까지 내 서버로 100% 자동 백업하실 수 있습니다!

[10단계] 안드로이드 스마트폰 앱 연동 및 사진 자동 백업

서버 구축의 진정한 꽃은 바로 '스마트폰 사진 자동 백업'입니다. 안드로이드 기기(주로 사용하시는 메인 스마트폰)에 Nextcloud 공식 앱을 설치하고 자동 업로드를 켜두면, 사진을 찍을 때마다 집에 있는 노트10+ 서버(SD카드)로 사진이 안전하게 전송됩니다. 이제 더 이상 클라우드 용량 부족 경고에 시달릴 필요가 없습니다!

서버 역할을 하는 노트10+가 아닌, 평소 들고 다니며 사용하는 메인 스마트폰에서 진행합니다.

  1. 구글 Play 스토어 앱을 엽니다.
  2. 검색창에 Nextcloud를 검색하여 파란색 동그라미 세 개가 그려진 공식 앱을 다운로드 및 설치합니다.

2. 서버 주소 입력 및 로그인

  1. Nextcloud 앱을 실행하고 [로그인]을 터치합니다.
  2. 서버 주소 입력창에 포트 번호(58080)를 포함한 완벽한 외부 접속 주소를 입력합니다.
⚠️ 서버 주소 입력 주의사항

반드시 https://로 시작해야 하며, 끝에 :58080 포트를 정확히 입력해야 합니다.

https://본인도메인.duckdns.org:58080
  1. 입력 후 화살표(다음) 버튼을 누르면 로그인 웹 화면이 뜹니다. 관리자 아이디와 비밀번호를 입력하고 [접근 허용 (Grant access)]을 터치합니다.

📱 자동 업로드 활성화 방법

  2. 메뉴 목록에서 [자동 업로드 (Auto upload)]를 선택합니다.
  3. 스마트폰 안의 여러 사진 폴더(카메라, 스크린샷, 카카오톡 등) 목록이 나타납니다. 백업을 원하는 폴더(예: Camera) 우측의 [구름 모양 아이콘]을 터치하여 파란색으로 활성화합니다.

4. 배터리 및 데이터 절약 세부 설정

  • Wi-Fi에서만 업로드: 데이터 요금이 걱정된다면 체크해 둡니다. 집에 와서 와이파이가 연결될 때만 사진이 백업됩니다.
  • 충전 중에만 업로드: 배터리 소모를 막기 위해 충전기를 꽂았을 때만 백업을 진행합니다.
  • 기존 파일도 업로드: 이 앱을 설치하기 전에 찍었던 과거의 사진들까지 싹 다 서버로 올리고 싶을 때 선택합니다.
  • 로컬 파일 유지/삭제: 서버로 전송이 완료된 사진을 스마트폰 기기에서 지워버릴지(용량 확보용), 아니면 폰에도 그대로 남겨둘지 결정합니다. 보통은 '기존 파일 유지'를 권장합니다.

[11단계] 가족 사진 백업 최적화 및 앨범 공유 설정

💡 상용 클라우드의 '패밀리 요금제' 완벽 대체!

이 단계에서는 스마트폰에서 서버로 사진이 백업될 때 년도별로 폴더를 자동 생성하여 깔끔하게 정리하고, 이름이 같은 파일이 업로드될 때 중복 여부를 판별하여 자동으로 이름을 변경(_1, _2)하는 스마트 백업 환경을 구축합니다. 또한, 가족 구성원 각자의 개인 사생활(개인 폴더)은 철저히 보호하면서 서로의 사진 앨범만 '조회(읽기 전용)' 할 수 있는 완벽한 가족 공유 클라우드를 세팅합니다.

1. 스마트폰 앱: 년도별 폴더 정리 및 중복 파일 처리 설정

가족들의 각 스마트폰에 설치된 Nextcloud 앱에서 '자동 업로드' 규칙을 아주 똑똑하게 수정해 줍니다.

  1. 스마트폰의 Nextcloud 앱 좌측 상단 [메뉴(가로줄 3개)] ➡️ [자동 업로드]로 들어갑니다.
  2. 카메라(Camera) 폴더 우측의 파란색 구름 아이콘 옆에 있는 [점 3개(설정) 아이콘]을 터치합니다.
⚙️ 자동 업로드 세부 설정 변경
  • 하위 폴더 사용 (Subfolders): 기본값인 '사용 안 함'을 눌러 [연도(Year)] 또는 [연도 및 월(Year and month)]로 변경합니다. 이제 서버에 사진이 올라갈 때 자동으로 `2026` 폴더가 생기고 그 안에 정리됩니다.
  • 파일이 이미 존재할 경우 (Existing file behavior): 이 부분을 [이름 바꾸기 (Rename)]로 선택합니다.
🤖 넥스트클라우드의 스마트 중복 처리 원리

넥스트클라우드 서버는 파일이 업로드될 때 고유의 지문(Hash)을 검사합니다. 만약 이름도 같고 지문(내용)도 완전히 똑같은 파일이 들어오면 서버 용량 낭비를 막기 위해 업로드를 취소(건너뛰기)합니다. 하지만 이름은 같지만 내용이 다른 사진(예: 다른 기기에서 찍은 동일한 이름의 사진)일 경우, 앱 설정에 따라 원본을 보호하기 위해 파일명 뒤에 자동으로 `_1`, `_2`를 붙여 새로운 파일로 안전하게 저장합니다.

2. 서버: 가족 구성원 개인 계정 및 그룹 생성

가족들이 각자의 독립된 공간을 가질 수 있도록, PC나 스마트폰 브라우저에서 '관리자 계정(admin)'으로 로그인하여 계정을 만들어 줍니다.

  1. 관리자 계정 로그인 후 우측 상단 프로필 ➡️ [사용자 (Users)] 메뉴로 이동합니다.
  2. 왼쪽 메뉴에서 [그룹 추가]를 눌러 Family라는 그룹을 하나 만들어 줍니다.
  3. [새 사용자] 버튼을 눌러 부모님, 배우자, 자녀 등의 아이디와 비밀번호를 만듭니다. 이때 '그룹' 항목을 방금 만든 Family로 지정해 줍니다. (할당량은 기기 용량에 맞게 무제한 또는 100GB 등으로 자유롭게 설정하세요.)

3. 갤러리 앨범만 안전하게 공유하기 (조회 전용)

이제 각 가족 구성원이 자신의 아이디로 로그인하여, 자신의 사진 폴더를 다른 가족들이 '볼 수만 있도록' 권한을 열어줍니다.

  1. 가족 구성원(예: 아빠)의 아이디로 넥스트클라우드 웹에 로그인합니다.
  2. 스마트폰에서 사진이 백업되고 있는 [Photos] 폴더(또는 사진이 모여있는 폴더) 우측의 [공유(🔗) 아이콘]을 클릭합니다.
  3. 우측 화면의 검색창('이름, 페더레이션 클라우드 ID 또는 이메일 주소 입력...')에 아까 만든 그룹 이름인 Family를 타이핑하여 선택합니다.
⚠️ 매우 중요: '편집 허용' 체크 해제!

Family 그룹이 추가되면 기본적으로 폴더에 대한 수정 권한이 부여됩니다. 다른 가족이 내 사진을 실수로 지우지 못하도록, 우측 설정에서 [편집 허용 (Allow editing)], [생성 (Create)], [삭제 (Delete)] 항목의 체크를 모두 해제하세요. 오직 [보기 가능 (Can view)]만 활성화되어야 합니다.

이제 다른 가족 구성원이 로그인하면, 메인 화면에 '아빠의 Photos'라는 폴더가 나타납니다. 들어가서 사진을 구경하거나 다운로드할 수는 있지만, 마음대로 지우거나 수정할 수는 없습니다. 완벽한 프라이버시와 공유가 결합된 나만의 패밀리 클라우드가 완성되었습니다!

[12단계] ipTIME 공유기 간이 NAS(외부 저장소) 완벽 연동

💡 스마트폰 전력 한계 극복 및 대용량 확장

스마트폰에 외장하드를 직접 연결하면 전력 부족으로 서버가 다운될 위험이 큽니다. 하지만 ipTIME 공유기의 USB 포트에 외장하드를 꽂고 네트워크(FTP)로 연결하면, 전원은 공유기가 부담하고 스마트폰은 통신만 담당하므로 24시간 무중단 대용량 서버를 가장 안전하게 구축할 수 있습니다.

1. 꽁꽁 숨겨진 '외부 저장소 지원' 앱 활성화하기

Nextcloud는 기본적으로 이 기능이 꺼져 있습니다. 관리자 계정으로 숨겨진 스위치를 켜주어야 합니다.

  1. PC 브라우저에서 관리자 계정(admin)으로 Nextcloud에 로그인합니다.
  2. 우측 상단 프로필 아이콘을 클릭하고 [앱 (Apps)] 메뉴로 들어갑니다.
  3. 왼쪽 사이드바 메뉴에서 [비활성화된 앱 (Disabled apps)]을 클릭하거나, 우측 상단 돋보기 아이콘을 눌러 External storage support를 검색합니다.
  4. 해당 앱을 찾았다면 [활성화 (Enable)] 버튼을 클릭합니다. (비밀번호를 한 번 더 물어볼 수 있습니다.)

2. ipTIME 공유기 설정 (FTP 서버 켜기)

이제 공유기에 외장하드를 꽂고, Nextcloud가 접속해서 파일을 넣고 뺄 수 있도록 통신문(FTP)을 열어줍니다.

  1. 공유기에 외장하드를 USB로 연결합니다.
  2. PC 브라우저 주소창에 192.168.0.1을 입력하여 ipTIME 관리자 페이지에 접속합니다.
  3. [고급 설정] ➡️ [USB/서비스 관리] ➡️ [장치 관리]로 들어가서 하드디스크가 정상적으로 인식되었는지 확인합니다.
  4. [고급 설정] ➡️ [USB/서비스 관리] ➡️ [서비스 설정] 메뉴로 이동합니다.
  5. 목록에서 [ipDISK 주소 관리]를 실행하여 적당한 주소와 이메일을 등록합니다.
  6. 가장 중요한 [FTP 서비스]를 클릭하고 [실행]으로 변경합니다.
  7. 하단의 사용자 ID와 암호를 입력하고, 권한은 [읽기/쓰기]로 선택한 뒤 [적용]을 누릅니다. (이 계정 정보는 다음 단계에서 쓰입니다.)

3. Nextcloud에 ipTIME 하드디스크 마운트(연결)하기

다시 Nextcloud 화면으로 돌아와서, 방금 만든 공유기의 FTP 창고를 내 클라우드에 묶어줍니다.

⚠️ SMB/CIFS 대신 FTP를 쓰는 이유

윈도우 공유 방식인 SMB(CIFS)는 안드로이드(Termux) 환경에서 호환성 충돌이 잦습니다. 가볍고 통신 규격이 확실한 FTP 방식으로 연결하는 것이 스마트폰 서버에서는 훨씬 안정적입니다.

  1. Nextcloud 우측 상단 프로필 아이콘 ➡️ [관리자 설정 (Administration settings)]을 클릭합니다.
  2. 왼쪽 관리(Administration) 메뉴 모음에 새롭게 생겨난 [외부 저장소 (External storages)]를 클릭합니다.
  3. 설정 칸을 다음과 같이 채워 넣습니다.
    • 폴더 이름 (Folder name): ipTIME_HDD (Nextcloud 화면에 표시될 폴더 이름입니다. 자유롭게 정하세요.)
    • 외부 저장소 (External storage): 드롭다운 메뉴에서 [FTP]를 선택합니다.
    • 인증 (Authentication): 로그인 정보 (Log in credentials)
    • 호스트 (Host): 192.168.0.1 (ipTIME 내부망 주소)
    • 사용자 이름 (Username): 아까 ipTIME 설정 2-7에서 만든 FTP 사용자 ID
    • 비밀번호 (Password): 아까 만든 FTP 암호
  4. 우측 끝에 있는 [V (체크 마크)] 버튼을 누릅니다.

🟢 녹색 동그라미가 떴다면 연결 성공!

입력창 왼쪽 끝에 초록색 원(🟢)이 표시되면 공유기의 외장하드와 Nextcloud가 완벽하게 한 몸이 된 것입니다! 이제 넥스트클라우드 [파일] 메뉴의 첫 화면으로 돌아가 보면 ipTIME_HDD라는 폴더가 생겨 있습니다. 이 폴더 안에 넣는 모든 자료는 스마트폰 용량을 1MB도 차지하지 않고, 공유기에 꽂힌 외장하드로 쏙쏙 저장됩니다.

🍯 꿀팁: 마운트 고급 옵션 (Check for changes) 설정법

외부 저장소 설정 창 우측의 [톱니바퀴 아이콘]을 누르면 변경 사항 확인(Check for changes) 주기를 설정할 수 있습니다.

  • Never (절대 안 함) - 🚀권장: 파일을 넣고 뺄 때 넥스트클라우드만 이용한다면 무조건 이 옵션을 선택하세요. 폴더 진입 속도가 비약적으로 빨라집니다.
  • Always (항상) / Once every direct access: 넥스트클라우드를 거치지 않고, 윈도우 PC에서 공유기 네트워크 폴더로 '직접' 파일을 넣는 일이 많을 때 선택합니다. 단, 폴더를 열 때마다 스캔을 하므로 로딩 속도가 조금 느려질 수 있습니다.
