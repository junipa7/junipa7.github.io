WSL2 & Windows IP 통합 가이드 (Mirrored Mode)

기본적으로 WSL2는 가상 네트워크(NAT)를 사용하므로 Windows와 IP가 다릅니다. 이 가이드는 Mirrored Networking Mode를 활성화하여 Windows의 IP 주소를 Ubuntu에서도 동일하게 사용하도록 설정하는 상세 과정을 설명합니다.

⚠️ 필수 요구 사양:
- OS: Windows 11 (버전 22H2 이상)
- WSL 버전: 최신 버전의 WSL (Microsoft Store에서 업데이트 권장)

1단계: .wslconfig 파일 생성 및 설정

WSL의 전역 설정을 담당하는 파일을 Windows 사용자 폴더에 만들어야 합니다.

  1. Win + R을 누르고 %UserProfile%을 입력하여 사용자 폴더를 엽니다.
  2. 폴더 빈 공간에서 우클릭하여 [새로 만들기] > [텍스트 문서]를 선택합니다.
  3. 파일명 전체를 .wslconfig로 바꿉니다. (끝에 .txt가 붙지 않도록 주의하세요.)
  4. 메모장으로 파일을 열고 아래 내용을 복사해서 붙여넣습니다.
[wsl2] # 네트워크 모드를 미러링 모드로 변경 networkingMode=mirrored # Windows 방화벽 설정을 WSL에도 자동으로 적용 firewall=true # Windows의 HTTP 프록시 설정을 공유 autoProxy=true # IPv6 사용 여부 (필요 시 true) ipv6=true

2단계: WSL 서비스 완전히 종료 및 재시작

설정 파일은 WSL이 완전히 꺼졌다가 다시 켜질 때 적용됩니다.

  1. Windows PowerShell을 관리자 권한으로 실행합니다.
  2. 다음 명령어를 입력하여 실행 중인 모든 WSL 인스턴스를 종료합니다.
wsl --shutdown

3. 다시 Ubuntu 아이콘을 클릭하여 실행합니다.

3단계: IP 주소 공유 확인

Ubuntu 터미널에서 실제로 Windows와 같은 IP를 사용하는지 확인합니다.

# Ubuntu 터미널에서 입력 ip addr show eth0

결과 화면의 inet 주소가 Windows PowerShell에서 ipconfig를 입력했을 때 나오는 IPv4 주소와 동일하다면 성공입니다.

4단계: 외부 접근을 위한 Windows 방화벽 설정

IP가 같아져도 Windows 방화벽이 막고 있으면 외부에서 접근할 수 없습니다. 웹 서버 포트를 열어주어야 합니다.

  1. 제어판 > 시스템 및 보안 > Windows Defender 방화벽 > 고급 설정으로 이동합니다.
  2. 왼쪽의 인바운드 규칙을 클릭하고, 오른쪽의 새 규칙...을 클릭합니다.
  3. 포트(O) 선택 → TCP 및 특정 로컬 포트(예: 80, 8080) 입력.
  4. 연결 허용 선택 → 도메인, 개인, 공용 모두 체크 → 규칙 이름(예: "WSL Web Server") 지정 후 마칩니다.
✅ 설정 완료 후 장점:
1. localhost 또는 Windows의 실제 IP로 어디서든 동일하게 웹 서버 접속 가능.
2. 별도의 복잡한 포트 포워딩(netsh) 명령어 불필요.
3. 네트워크 성능 및 호환성 향상.
