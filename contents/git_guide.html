WSL2 Ubuntu 환경 Git 등록 상세 가이드

이 가이드는 기존의 잘못된 설정을 제거하고, 처음부터 GitHub에 소스를 올리는 전 과정을 담고 있습니다.

1단계: 기존의 잘못된 Git 설정 완전 삭제

기존에 꼬여있던 .git 폴더를 삭제하여 초기 상태로 만듭니다. (소스 파일은 삭제되지 않습니다.)

# 작업 폴더로 이동
cd ~/your-project-folder

# 기존 Git 설정 강제 삭제
rm -rf .git

2단계: Git 사용자 정보 설정 (Identity)

커밋을 남길 때 사용할 사용자 이름과 이메일을 Ubuntu 시스템에 등록합니다.

# 이메일 등록 (GitHub 이메일 권장)
git config --global user.email "your-email@example.com"

# 이름 등록 (영문 권장)
git config --global user.name "YourName"

# 설정 확인
git config --list

3단계: GitHub Personal Access Token (PAT) 발급

보안 정책상 비밀번호 대신 토큰을 사용해야 합니다. GitHub 웹사이트에서 수행하세요.

  1. GitHub 우측 상단 프로필 클릭 → Settings
  2. 왼쪽 메뉴 하단 Developer settings 클릭
  3. Personal access tokens → Tokens (classic) 클릭
  4. Generate new token (classic) 선택
  5. Note: wsl-access-token 등 이름 입력
  6. Expiration: 기간 선택 (예: 30 days)
  7. Select scopes: repo 항목에 반드시 체크 (저장소 접근 권한)
  8. 하단 Generate token 클릭 후 생성된 문자열을 반드시 복사하여 따로 보관하세요.

4단계: GitHub 원격 저장소 생성

온라인상에 파일을 담을 빈 저장소를 만듭니다.

  1. GitHub 메인에서 New Repository 클릭
  2. Repository name 입력 (예: my-project)
  3. Public 선택 후, Create repository 클릭
  4. 생성된 페이지의 HTTPS 주소(https://github.com/...)를 복사합니다.

5단계: 로컬 소스 코드 초기화 및 업로드

이제 Ubuntu 터미널에서 실제로 파일을 전송합니다.

# 1. 현재 폴더를 Git 저장소로 지정
git init

# 2. 모든 파일을 업로드 대기 상태로 추가
git add .

# 3. 첫 번째 커밋 생성
git commit -m "First commit: Initialize project"

# 4. 기본 브랜치명을 main으로 변경
git branch -M main

# 5. GitHub 주소 연결
git remote add origin [복사한_GitHub_주소]

# 6. 서버로 전송
git push -u origin main
로그인 요청 시:
- Username: GitHub 아이디 입력
- Password: 위 3단계에서 복사한 토큰(Token)을 붙여넣기

6단계: 인증 정보 저장 (토큰 재입력 방지)

앞으로 토큰을 다시 입력하지 않도록 인증 정보를 저장합니다.

# 파일에 영구적으로 인증 정보 저장 설정
git config --global credential.helper store

이후 한 번만 더 git push를 하여 인증을 완료하면 ~/.git-credentials 파일에 저장되어 다음부터 묻지 않습니다.
