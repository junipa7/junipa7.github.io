2 6대 기본 상태와 세부 상태 (Sub-states)

각 장비 상태의 정확한 정의와, 해당 상태에 속하는 세부 작업들을 탐색합니다.

3 RAM 핵심 지표와 OEE 연계

가상의 주간(168시간) 데이터를 바탕으로 장비 성능 KPI가 어떻게 계산되는지 확인합니다.

가상 시간 분포 (총 168시간)

차트 조각에 마우스를 올리면 세부 시간을 확인할 수 있습니다.

PRD: 100h
SBY: 20h
ENG: 10h
SDT: 15h
UDT: 15h
NST: 8h

신뢰성 및 보전성 (고장 횟수 = 5회 가정)

MTBF (평균 무고장 시간) 20.0 시간
PRD (100h) / Failures (5)
* 전체 가동 시간이 아닌 순수 생산(PRD) 시간 기준
MTTR (평균 수리 시간) 3.0 시간
UDT (15h) / Failures (5)
* 고장 인지, 대기, 수리, 검증 완료까지의 총합

가용성 및 활용도 (OEE 연계)

가용성 (Equipment Availability) 81.3%
(Uptime / Ops Time) * 100 = (130 / 160) * 100
* 이 값이 SEMI E79 OEE 공식의 '가동률(A)'이 됩니다.
운영 활용도 (Operational Utilization) 62.5%
(PRD / Ops Time) * 100 = (100 / 160) * 100
* 가용성이 높아도 SBY가 길면 활용도는 떨어집니다.

4 실무 적용 및 비즈니스 룰

데이터 자동 수집 환경에서 겪게 되는 애매한 상황들에 대한 표준 처리 가이드입니다.

복합 상태 (Overlapping States) 우선순위

한 순간에 2개의 상태가 겹칠 때 어떤 상태로 기록해야 하는가?

  • 예시 1: SBY 중 UDT 발생 자재를 기다리며 쉬고(SBY) 있었는데, 정전으로 장비가 꺼짐(UDT).
    ➔ UDT (고장/오프라인이 대기보다 우선함)
  • 예시 2: SDT 중 UDT 발생 계획된 예방정비(SDT) 중 엔지니어 실수로 부품이 파손되어 수리 연장.
    ➔ 원래 계획 시간까지는 SDT, 초과 시간부터는 UDT 전환

Failure(고장) vs Assist(어시스트) 구분

E10에서 가장 논란이 많으며 팹마다 기준이 다르나, 일반적인 가이드는 다음과 같습니다.

구분 Failure (UDT 처리) Assist (단기 중단)
복구 주체 전문 정비 엔지니어 일반 오퍼레이터/자동 리셋
조치 내용 부품 교체, 장비 분해 버튼 클릭, 걸린 웨이퍼 빼기
소요 시간 보통 6분(또는 10분) 초과 수 분 이내에 복구 가능
영향도 재가동 전 테스트 수반 즉시 원래 작업 재개
